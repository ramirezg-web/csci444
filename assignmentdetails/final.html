Drawing on the Web

Your final project will consist of three original, full-browser drawings and a custom interface to access them. You will incorporate multiple techniques for drawing on the web including vector and/or raster graphics, duration, interactivity, and data. The project has three parts to it: a project outline, the project itself, and a presentation. Optionally, you may choose to work in pairs.

Outline

Thought Process

Before you begin working on your final project, take some time to consider how you would like to approach it. What kind of experience do you want to elicit? How will the three drawings relate to each other aesthetically and/or conceptually? What kind of interactivity will there be and how will you achieve it?

Text and Prototype

Write a concise project outline of at least 200 words describing your intent. Consider the outline well; your final project should clearly emerge from what you describe in the text. Then, prototype a version of one of the drawings—with code, in the browser. This drawing can evolve with your project but represents the general direction of your work. If you will be working with a partner, you will need to write the outline and create the prototypal drawing collaboratively.

Submitting Your Outline

The project outline is due before class on Wednesday, April 20. Submit your text and prototype via NYU Brightspace to the “Final Project Outline” assignment. Copy and paste the text of your outline into the text box, upload the files of your first drawing, and include a link to your prototype on i6. If you work in pairs, you should both submit a copy of the outline via Brightspace and indicate your collaborator.

Project

Three Drawings

The final project is an opportunity for you to develop a series of three original web drawings with the coding techniques we’ve been exploring all semester. These drawings are intended to be seen and experienced more than read; as such, they should should occupy the full browser window and you should keep HTML text to a minimum. Each of the drawings should include at least one raster or vector graphic that is rendered in the browser (such as SVG, the HTML canvas, or CSS shapes). Your drawings should include durational change in the form of CSS transitions/animation, JavaScript DOM animation, or canvas animation. And there should be some aspect of JavaScript-based user interactivity to the pages.

Navigational Interface

In addition to the three separate web pages for each drawing, create a fourth page that will serve as an interface to access them. You should be as thoughtful and creative in designing the interface as you are with the drawings themselves. The primary requirement for the interface is that it provide links to navigate to each of the three drawing pages. When the project is published, add a link from your Drawing on the Web assignments directory to the navigational interface.

Web API

Your navigation page should have an element of variability based on data drawn from a web API. Consider the relationship between your project and the API you work with as well as how its data is translated into the page. You’re welcome to work with the same API you explored in the assignment; this page includes other APIs that the class researched. Your individual drawing pages do not require an API call as the navigational interface does.

State of Completion

By the time you present, your project should be fully complete. Your CSS and JavaScript should be linked to as external documents and the pages should be viewable in current versions of contemporary web browsers. As you finish your work, pay attention to presentational aspects of the site and the overall quality of its display. Although there are several required elements, your project does not need to be highly complex; it should, however, be polished. Moreover, the concept and code implementation must be original and unique to this project.

Presentation

You will be required to do a brief presentation of your work on Wednesday, May 4. This is a time for you to share your accomplishments and describe your process. Your presentation should be approximately 3 minutes in length and well-prepared. As with the project outline, the presentation is part of your grade. Consider how you will introduce, describe, and demo your work and also talk about its technical development.

Submitting Your Project

Publish your project on the i6 web server, update the link from your portfolio page, and submit the following via NYU Brightspace.

  • The URL to your project in the general form of: i6.cims.nyu.edu/~netid/drawing/
  • The URL to the web API used on your navigation page
  • A compressed archive of your project files (including the HTML, CSS, JavaScript, and any media resources)

Please note that you are required to submit files along with URLs in order to receive credit for your work.

Grading

The final project is worth 30 points.

Outline
  • Thoughtfully-considered project outline of at least 200 words (3 points)
  • A first version of one of the drawings (2 points)
Project
  • A series of three different web drawings that occupy the full browser window (6 points)
  • Each of the drawings should include at least one raster or vector graphic that is rendered in the browser (3 points)
  • Each of the drawings should include durational change in the form of CSS transitions/animation, JavaScript DOM animation, or canvas animation (3 points)
  • Each of the drawings should have some aspect of JavaScript-based user interactivity (3 points)
  • A thoughtfully-designed navigation page with links to access the drawing series (3 points)
  • The navigation page should have an element of variability based on data drawn from a web API (2 points)
  • Project should be fully complete with well-formed HTML, CSS, and JavaScript, cross-browser compatibility, and an organized file structure (2 points)
Presentation
  • Well-prepared in-class presentation, approximately 3 minutes in length, of project concept and development (3 points)
